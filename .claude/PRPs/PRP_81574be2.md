# Propuesta de Requisitos del Producto (PRP)
## Proyecto: 81574be2-6583-4f83-9786-fe2daf54d7e7
### Objetivo
El objetivo del proyecto es desarrollar una aplicación web que permita a los usuarios gestionar y organizar sus tareas de manera eficiente. La aplicación debe ser fácil de usar, escalable y segura.

### Alcance
El alcance del proyecto incluye:
* Desarrollo de la aplicación web
* Diseño de la interfaz de usuario
* Implementación de la base de datos
* Desarrollo de los endpoints API
* Integración de la autenticación y autorización
* Pruebas y depuración

### Stack Técnico
El stack técnico utilizado para el proyecto será:
* Lenguaje de programación: Python 3.9
* Framework: Flask 2.0
* Base de datos: PostgreSQL 13
* Frontend: React 17.0
* Bibliotecas adicionales: SQLAlchemy, Flask-SQLAlchemy, React-Bootstrap

### Tablas de DB
Las tablas de la base de datos serán las siguientes:

| Tabla | Descripción |
| --- | --- |
| usuarios | Almacena la información de los usuarios |
| tareas | Almacena la información de las tareas |
| categorias | Almacena la información de las categorías de tareas |

Ejemplo de estructura de las tablas:
```sql
CREATE TABLE usuarios (
    id SERIAL PRIMARY KEY,
    nombre VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE,
    contraseña VARCHAR(255) NOT NULL
);

CREATE TABLE tareas (
    id SERIAL PRIMARY KEY,
    titulo VARCHAR(100) NOT NULL,
    descripcion TEXT,
    fecha_creacion TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    fecha_vencimiento TIMESTAMP,
    usuario_id INTEGER NOT NULL REFERENCES usuarios(id)
);

CREATE TABLE categorias (
    id SERIAL PRIMARY KEY,
    nombre VARCHAR(50) NOT NULL,
    descripcion TEXT
);
```

### Endpoints API
Los endpoints API serán los siguientes:

| Endpoint | Método | Descripción |
| --- | --- | --- |
| /usuarios | GET | Obtiene la lista de usuarios |
| /usuarios | POST | Crea un nuevo usuario |
| /usuarios/:id | GET | Obtiene la información de un usuario |
| /usuarios/:id | PUT | Actualiza la información de un usuario |
| /usuarios/:id | DELETE | Elimina un usuario |
| /tareas | GET | Obtiene la lista de tareas |
| /tareas | POST | Crea una nueva tarea |
| /tareas/:id | GET | Obtiene la información de una tarea |
| /tareas/:id | PUT | Actualiza la información de una tarea |
| /tareas/:id | DELETE | Elimina una tarea |

### Componentes UI
Los componentes UI serán los siguientes:

* Barra de navegación
* Lista de tareas
* Formulario de creación de tareas
* Formulario de edición de tareas
* Botón de eliminación de tareas

### Criterios de Aceptación
Los criterios de aceptación serán los siguientes:

* La aplicación debe ser capaz de crear, leer, actualizar y eliminar tareas
* La aplicación debe ser capaz de crear, leer, actualizar y eliminar usuarios
* La aplicación debe ser capaz de asignar tareas a usuarios
* La aplicación debe ser capaz de filtrar tareas por categoría
* La aplicación debe ser capaz de ordenar tareas por fecha de vencimiento
* La aplicación debe ser capaz de mostrar una lista de tareas pendientes
* La aplicación debe ser capaz de mostrar una lista de tareas completadas

La aplicación debe cumplir con los siguientes estándares de calidad:

* La aplicación debe ser escalable y segura
* La aplicación debe ser fácil de usar y entender
* La aplicación debe ser compatible con diferentes navegadores y dispositivos

La aplicación debe ser entregada con la siguiente documentación:

* Guía de instalación
* Guía de usuario
* Documentación de la API
* Documentación de la base de datos

La aplicación debe ser entregada en un plazo de 12 semanas, con las siguientes hitos:

* Semana 1-2: Desarrollo de la base de datos y la API
* Semana 3-4: Desarrollo de la interfaz de usuario
* Semana 5-6: Integración de la autenticación y autorización
* Semana 7-8: Pruebas y depuración
* Semana 9-12: Entrega de la aplicación y documentación
